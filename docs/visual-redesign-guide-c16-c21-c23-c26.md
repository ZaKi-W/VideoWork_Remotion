# 口播摘要组件（C16、C21、C23-C26）视觉重构与无界悬浮设计指南

本指南用于指导后续介入项目的 AI 代理（Agents）或开发人员，了解本次对口播摘要组件系列所进行的**极致无界融入与纵向画幅拉伸**的重构设计细节。

---

## 1. 重构背景与设计初衷

在旧版本中，口播摘要组件（C16、C21、C23-C26）存在两个关键痛点：
1. **不融入视频背景**：组件使用了深色实色盒子背景或过重的黑色暗化 Scrim，悬浮于口播视频上方时会呈现出明显的“生硬色块边界”，阻断了人像和视频画面的流畅度。
2. **纵向比例偏小局促**：组件在高度上占比太小（仅占屏幕的 40%~50%），当用户一边听口播一边阅读时，文字拥挤、折行严重，阅读体验差。

针对以上痛点，我们执行了**“极轻量无界 HUD 悬浮”**与**“纵向大画幅拉伸”**的全新美学设计。

---

## 2. 核心重构设计原则

后续对这些组件进行样式或动效修改时，必须严格遵守以下原则：

### 🟢 极致透底与 HUD 悬浮感
*   **移除所有深色背景色块**：除全屏背景视频模式外，最外层容器背景完全设为 `transparent`，子卡片改用极低透明度的白色（`rgba(255, 255, 255, 0.02 - 0.05)`），使视频画面自然无碍地透出。
*   **工业对焦十字与拐角（Reticle & Corners）**：使用 `1px` 宽的半透明白定位线、直角拐角和 `+` 字准心符号充当视觉定位骨架，呈现全息透镜（HUD）悬浮感。
*   **环境光流（Light Leak）**：用极高模糊（`blur(90px)`）、极低透明度（`4% - 5%`）的强调色粒子，在卡片内部漫反射流动，像真实舞台的霓虹溢光一样叠加在视频上，绝不用硬边框渐变色。

### 🔴 字幕安全与纵向最大化排版
*   **纵向占比提升**：统一将组件的物理 top 起点从原先的 `129px - 151px` 提升到顶边距离顶部 **`7.5%`（约 `81px`）**。将外容器最小高度 `minHeight` 增加到 **`73%`（即 `788px`）**。
*   **字幕安全区（Subtitle Safety Cushion）**：
    *   字幕区位于底部 `18%`（即 `y >= 885px`）。
    *   重构后，组件底部最高 `y` 坐标为 `81 + 788 = 869px`。与安全区上边界（`885px`）之间保有 **`16px` 的防侵入安全距离**。任何新加的文字和子项严禁超出该下限。
*   **拉开段落间距**：在大纵向画幅下，将 kicker、title、copy 和 foot 之间的 `marginTop` 从 12-18px 放宽到 **`32px - 40px`**，呈现高端大气的呼吸感。

### 🔵 全局投影与可读性保障
*   在去底板透底后，为了防止白色正文与浅色或高亮的口播视频背景发生粘连、导致无法认读，所有文字组件都必须附加**复合黑阴影保护**：
    `textShadow: '0 2px 16px rgba(0, 0, 0, 0.72), 0 1px 4px rgba(0, 0, 0, 0.6)'`。

## 4. 技术实现细节

*   **局部 Keyframes 注入**：
    由于 React Inline Styles 无法直接定义动画帧，我们设计了自包含的 `<StyleEffects />` 虚拟组件，在组件顶层通过 `<style dangerouslySetInnerHTML={{...}} />` 注入动画帧，这使得重构后的组件在 Remotion 环境中极为稳健：
    ```typescript
    // 例：C16 内的 StyleEffects 声明
    const StyleEffects = ({accentColor}: {accentColor: string}) => {
      const cleanColor = accentColor.replace('#', '');
      return (
        <style dangerouslySetInnerHTML={{__html: `
          @keyframes rippleWave-${cleanColor} {
            0% { transform: scale(0.95); opacity: 0.85; }
            100% { transform: scale(3.0); opacity: 0; }
          }
          @keyframes cursorBreathe {
            0%, 100% { opacity: 1; transform: scaleY(1); }
            50% { opacity: 0.15; transform: scaleY(0.9); }
          }
        `}} />
      );
    };
    ```

*   **打字机的物理阻尼与爆破发光**：
    打字机字符出现时，通过 `rangeProgress` 捕捉帧插值，使用以下贝塞尔曲线控制弹动缩放和位移，并利用 `textShadow` 控制第 2-3 帧时瞬间亮起的发光效果：
    ```typescript
    const scale = interpolate(charProgress, [0, 1], [1.2, 1], {
      easing: Easing.bezier(0.175, 0.885, 0.32, 1.15)
    });
    const glowPhase = interpolate(charProgress, [0, 0.4, 1], [0, 1, 0]);
    const textShadow = glowPhase > 0.1 ? `0 0 ${glowPhase * 16}px ${accentColor}` : '0 2px 14px rgba(0,0,0,0.72)';
    ```

## 5. 校验与日常维护

每次修改此系列组件时，必须在工作区下运行以下命令进行自动化 QC 检查：
1. **类型检查**：`npm run typecheck`
2. **风格检查**：`npm run lint`
3. **单元测试**：`npm run test`
4. **Studio 预览**：`npm run dev`，打开浏览器进入 `localhost:3000` 观察实际渲染动画表现。
