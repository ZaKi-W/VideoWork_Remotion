# Visual System

视觉体系名称：白底科技编辑部 / AI 纸面实验室。

原则：

- 白墙实拍人物与纸白资料底保持统一。
- 炭黑文字承担主体信息。
- 信号橙只用于关键强调、警示和当前结论。
- 电蓝只用于流程、技术路径、步骤和引用索引。
- 人物居中时，普通信息组件不得进入人物安全区。
- 所有正文信息不得进入字幕安全区。
- 大字结论优先于普通卡片。
- 组件应像杂志排版、批注、索引、剪报、编号或数据图，而不是游戏 HUD。

禁止默认使用：

- 赛博朋克霓虹。
- HUD 战报面板。
- 扫描线、雷达、粒子。
- 大量发光描边。
- 玻璃拟态。
- 奶油网页卡片风。
- 固定“左面板 + 中间人物 + 右侧新闻卡”的布局。
- 为了炫技而出现的无意义动画。

公共组件标准：

- 至少预计复用两次。
- 有明确的叙事任务。
- 有 props schema、注册表状态和校验规则。
- 不依赖单期素材或单期文案。

## SectionStamp

`SectionStamp` 是强势章节标题，不再是轻量索引条。它用于话题切换时快速建立视觉重心：黑色超粗中文标题、一个高对比色块、极简编号或小标签。

- 标题不是栏目标签，而是章节主视觉。
- 默认使用短标题、两行、超粗、黑白高对比、一个强调词、一个强调色块。
- 辅助信息必须弱于标题，总面积保持很小。
- `impact` 适合 `top-left` / `top-right`。
- `edge-impact` 适合 `edge-left` / `edge-right`。
- 信号橙是默认强调色。
- 电蓝只用于技术、流程、步骤类章节。
- 不使用大圆角白卡、HUD 框、霓虹、扫描线、发光描边、传统电视栏目包装或复杂资料页边签。

## HeadlineTakeover

`HeadlineTakeover` 是观点重锤：用于一段里的最强观点、结论、反转、关键判断或章节记忆点。它不是章节入口，不显示编号、栏目标签、英文索引、边签或多层辅助信息。

- 黑色超粗大字是唯一主角，默认 1 到 3 行。
- 视觉元素控制为大标题、一个强调词、极少量色块。
- `punch` 从一侧压入，适合人物居中时避开脸部、上半身和手势区。
- `wrap` 围绕人物安全区形成非对称包围感。
- `takeover` 只用于 `no-presenter` 或 `screen-primary`，允许画面被标题短暂接管。
- 强调色只用一种：信号橙默认，电蓝只用于技术、Agent、工作流、工程语境。
- 不适合普通转场、每一句话、普通标题、新闻/官网/录屏/数据说明，也不要与 `SectionStamp` 连续叠用。
- 一条 5 到 10 分钟视频建议 2 到 5 次，两次之间尽量间隔 20 到 40 秒有效内容。

## ConceptSplit

`ConceptSplit` 是 Editorial Contrast Cut / 编辑对照切片：用于解释两个概念、两种方式、两个阶段之间的本质差异。它不是章节入口、观点重锤、证据剪报或数据账页。

- 旧概念更克制、更灰、更像被替代的旧状态。
- 新概念更粗、更黑、更靠近强调色，视觉重量必须明显更强。
- 默认表达从旧方式到新方式、从聊天到执行、从工具到流程的认知转折。
- `cross-cut` 用于人物仍在画面中时，概念词围绕人物安全区形成不对称切片。
- `editorial-fold` 用于无人物或屏幕主视觉，两个纸面区域沿折页切口相遇。
- `handoff` 用于工具演示或流程讲解，表达一次关键跃迁。
- 强调色只用一种：信号橙默认，电蓝只用于 Agent、自动化、代码、工程、工作流、技术能力。
- 不使用左右对称面板、两个规整白卡、传统 PPT 左右栏、红蓝阵营、HUD、霓虹、扫描线、粒子或大面积网格。
- 一条 5 到 10 分钟视频建议 2 到 5 次，两次之间插入口播、数据、证据、录屏或案例；连续两次必须更换 `mode` 或 `anchor`。

## EvidenceClip

`EvidenceClip` 是证据剪报：官网公告、新闻、推文、价格页、报告局部、官方说明文档或研究图表的来源素材展示。它必须绑定 `assetId` 与 `sourceRefId`，不能伪造网页、新闻、推文或数据图。

- `clipping` 是辅助证据：资料从边缘抽出，局部裁切，带一个重点高亮和最多一个辅助批注。
- `spotlight` 是主证据：资料短暂接管画面，但仍保留字幕安全区和小型来源索引。
- 截图边界必须清楚，使用深灰细边、轻微纸面层次或阴影，避免白底截图融进白墙。
- 来源标识像档案索引条，足够小但可读，不做底部粗黑 `SOURCE` 条。
- 高亮来自 scene 配置坐标，不由 AI 猜测重点。
- 信号橙用于重点结论、价格、变化、关键句。
- 电蓝用于技术流程、代码、Agent、工具能力。
- 不做右侧巨大白色圆角网页卡，不做固定“左信息板 + 中间人物 + 右网页卡”，不做新闻播报模板。
- 没有可靠来源时，改用抽象解释型视觉，不使用 `EvidenceClip`。

## MetricSpread

`MetricSpread` 是 Data Ledger / 数据账页：用于一个主指标和少量前后对照数据。它必须绑定 `sourceRefId`，不展示完整网页截图，不承担证据图批注。

- 主数字是真正主角，应超粗、紧凑、黑白高对比。
- 数据行像账页记录，不像普通表格、Excel 截图或财经节目图表。
- 一次只突出一个值，辅助信息总视觉面积弱于主数字。
- 比率条最多一条，只暗示变化，不做柱状图或仪表盘。
- 来源索引是小型资料索引，不做黑色 `SOURCE` 底栏。
- 信号橙用于成本、价格、关键效率变化和结论性数字。
- 电蓝只用于 Agent、工作流、自动化、工程速度、技术能力。
- 辅助型 `MetricSpread` 只放左侧：`top-left` 或 `edge-left`。右侧 slot 已禁用。
- 不使用暗色战报、HUD 外框、赛博元素、渐变、发光、扫描线、大圆角白卡或密集小字。
- 需要展示价格页或报告截图时，用 `EvidenceClip`；需要整理关键数字时，用 `MetricSpread`。
