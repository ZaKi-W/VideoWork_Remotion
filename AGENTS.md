- 这是一个 Remotion 视频生产工程。
- 视频生产必须遵循：分镜 → 素材确认 → scene 配置 → 关键帧 QC → 启动 Remotion Studio/网页预览 → 用户确认 → 按需低清预览或高清渲染。
- 每次生成视频特效时，不允许默认直接渲染导出视频；必须先把项目启动起来，提供网页预览地址，让用户先在浏览器里看效果。
- 只有用户明确要求导出、低清预览文件或高清渲染时，才可以执行 `episode:preview`、`episode:render`、`remotion render` 或其他会生成视频文件的命令。
- 不允许直接按 SRT 每句话随机堆特效。
- 一段口播只承担一个主要视觉任务。
- 每段最多一个主组件，加零到两个辅助标注。
- 人物居中时，不允许普通组件进入人物安全区。
- 所有模式下，不允许正文信息进入字幕安全区。
- EvidenceClip 等证据类画面必须绑定可追溯来源。
- 没有来源的数据、新闻、网页内容不得伪装成事实证据。
- 无法确认来源时，只能使用抽象解释型视觉。
- 预计未来不能复用至少两次的效果，不得进入公共组件库。
- 一次性效果应留在具体 episode 内。
- 所有新增组件都必须分配稳定编号，写入 `src/editorial/registry/component-catalog.ts`，并注册到 Remotion Studio 左侧 Composition 菜单，确保可以像现有组件一样单独打开查看。
- 每次针对组件表现、样式、动效、布局或行为的修改，都必须优先改组件本体，确保所有使用该组件的 episode / demo 同步生效；不得只在某一个 episode 配置里做局部覆盖，除非用户明确要求只改该期。
- 每次任务执行都不要进行深度测试，除非用户明确要求深度测试。
- 每次改动后必须执行相关 lint、typecheck、test 或 smoke test。
