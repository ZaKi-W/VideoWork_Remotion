组件系统原则：多样化，但由内容触发

6 类组件都保留。它们不是互斥风格，而是同一频道视觉语言下的不同“语法”。每段口播只选一个主组件，最多两个辅助标注。

出现什么元素，就用什么组件

网页 / 新闻 / 报告 / 截图来源
用 来源证据 Dock。来源必须可追溯；不确定来源时，只能做抽象解释框。
步骤 / 清单 / 方法 / 结论
用 实体卡片栈。一张卡承担一个信息点，靠切入和错层产生节奏。
模型链路 / Agent 流程 / 架构关系
用 模型结构浮层。节点连线保持抽象，不伪造具体数据或品牌事实。
产品 UI / 参数 / 截图局部 / 功能点
用 扫描式标注。扫描线只服务一个焦点，不把界面拆成一堆随机特效。
A/B 工具 / 两种路线 / 优缺点
用 工具对比天平。左右信息保持对称，中间让给人物判断。
观点 / 争议 / 判断 / 风险
用 观点拆解浮层。主张、依据、风险分层出现，适合行业判断视频。

统一视觉约束

同一套材质 暗底、半透明黑玻璃、白色主信息、青绿/金色少量强调，避免紫蓝 AI 模板感。
同一套空间纪律 人物居中时，普通组件不进人物安全区；正文信息不进字幕安全区。
同一套动效语法 精准切入、短暂停顿、轻微物理回弹、快速收束；不靠满屏粒子和随机漂浮。
同一套信息密度 一个主组件最多承载一个主要视觉任务，辅助标注只解释或定位，不抢叙事。
